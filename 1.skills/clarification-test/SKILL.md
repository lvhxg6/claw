# 澄清确认测试 Skill

这是一个用于测试 `ask_clarification` 工具的 Skill，模拟真实业务场景中需要向用户确认或补充信息的情况。

## 测试场景

### 场景 1: 信息不足时请求补充

当用户说"帮我操作服务器"但没有提供具体信息时：

**必须使用 `ask_clarification` 工具询问：**
```
问题: 请提供以下信息以便执行服务器操作：
选项:
- 提供服务器 IP 地址
- 提供服务器主机名
- 查看可用服务器列表
```

### 场景 2: 高风险操作前确认

当用户要求执行以下操作时，**必须先使用 `ask_clarification` 确认**：
- 重启服务
- 删除文件
- 修改配置
- 执行加固

**确认示例：**
```
问题: 即将执行 [操作名称]，此操作可能影响服务可用性。是否继续？
选项:
- 确认执行
- 取消操作
- 查看操作详情
```

### 场景 3: 多选项场景

当用户说"帮我检查服务器"时：

**使用 `ask_clarification` 询问检查类型：**
```
问题: 请选择要执行的检查类型：
选项:
- 基线检查
- 弱口令检查
- 全部检查
- 自定义检查项
```

### 场景 4: 开放式问题

当用户的需求模糊时（如"帮我处理一下"）：

**使用 `ask_clarification` 但不提供选项：**
```
问题: 请详细描述您需要处理的具体内容，包括：1) 操作对象 2) 期望结果 3) 任何特殊要求
选项: null（不提供预设选项，让用户自由输入）
```

## 工作流程

1. **接收用户请求**
2. **判断信息是否充足**
   - 如果不充足 → 使用 `ask_clarification` 补充信息
3. **判断操作风险级别**
   - 如果是高风险操作 → 使用 `ask_clarification` 请求确认
4. **执行操作**（本测试 Skill 只模拟，不实际执行）
5. **返回结果**

## 测试用例

用户可以尝试以下输入来测试不同场景：

| 用户输入 | 预期行为 |
|---------|---------|
| "帮我操作服务器" | 询问服务器信息 |
| "重启 192.168.1.1 的服务" | 确认是否执行高风险操作 |
| "检查服务器" | 询问检查类型 |
| "帮我处理一下" | 开放式询问具体需求 |
| "检查 192.168.1.1 的基线" | 直接执行（信息充足且非高风险） |

## 注意事项

- 这是一个测试 Skill，不会执行实际的服务器操作
- 主要用于验证 `ask_clarification` 工具的交互流程
- 用户回复后，Agent 应根据回复内容继续处理或再次询问
