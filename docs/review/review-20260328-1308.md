## 修改建议

1. 把 `artifact_middleware / governance_middleware / step_tracking_middleware` 抽成显式接口层。
   最小做法不是重写 graph，而是在 [orchestrator_graph.py](/Users/liubu/hx/hxWork/35.AI测试/claw/smartclaw/smartclaw/agent/orchestrator_graph.py) 里加统一 middleware runner，把现有 `normalize/review/dispatch` 内逻辑迁进去。

2. 把 skill 从“runtime 全量加载”改成“按 plan/step 的 `preferred_skill` 或 capability pack 选择后再加载”。
   这样才真正符合文档要的“按需加载”，也能避免系统 prompt 和工具表继续膨胀。

3. 增加 dispatch 前审批 gate。
   已有 `approval_required` 字段和 step 风险推导，下一步应让 orchestrator 在 `dispatch before` 对审批状态做硬拦截，而不是只把字段带着走。

4. 把 replanning 的止损语义补齐成统一策略。
   至少补 `repeated_error_guard` 和“预算耗尽后返回当前可交付结果 + 阻塞原因”的 fallback 输出。
